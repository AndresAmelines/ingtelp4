Para el caso del campus el procedimiento es similar, a diferencia que ya no se usaría IPv4, sino IPv6, siendo más amplio para cubrir una mayor cobertura de dispositivos conectados e inclusive una mayor seguridad por la gran cantidad de dígitos y letras que se designa por dispositivos.

Ejemplo de edificación por bloque:


Vemos un ejemplo de la magnitud de la IPv6, el cual se podría designar inclusive de una dirección para cada bloque y de cada una desprender la cantidad de subredes que se desee. En número de direcciones que abarca es: 3.4 x 10 a la potencia de 38, equivalente a un trillón de direcciones para cada ser humano en el planeta.


El direccionamiento de la IPv6 inicia desde 0000 hasta FFFF. Es decir que para aplicarlo al campus una opción podría ser designar una IPv6 para cada bloque de la siguiente manera.

Tomando como ejemplo la imagen anterior, se opta porque las direcciones comiencen con ceros para abreviarlos y reducir la dirección visualmente, teniendo en cuenta la regla de 3-1-4, el cual significa que los tres primeros segmentos son globales, es decir, que no se deben cambiar, y utilizar el cuarto o incluso más segmentos para distribuir las subredes.

Aclaración: Estos tres primeros segmentos se define según la ubicación geográfica del mundo y según las empresas que ofrecen servicios de internet, ejemplo del IP por el servicio de internet que tenemos: 2001:BA11:B011.

Se crea la siguiente red para el primer edificio:

• 2001:BA11:B011:1000:0001:F001:F003:0001

De este se desprenden las subredes necesarias para suplir las necesidades, dándole una capacidad de mil, lo que se considera suficiente para un edificio y la independencia de la facultad frente a las otras, para el segundo edificio solo cambia el número que está en rojo.

• 2001:BA11:B011:2000:0001:F001:F003:0001

Y así sucesivamente hasta llenar los diez bloques del campus universitario.

En este ejemplo observamos que de las mil direcciones IPv6 que tiene el edificio se pueden asignar de 100 para cada piso para identificar más fácil el número de IP por edificio y por piso, y así sucesivamente para los nueve edificios restantes y con los dispositivos, siendo controlados por Switch.

Se hace uso del método EUI– 64 para asignar direccionamientos IPv6 de manera automática, haciendo uso del IP de la MAC del equipo, comprendiendo que son muchísimos dispositivos y manualmente tardaríamos demasiado.
